Skills Manager is for people who already have a mess of scattered `SKILL.md`, `AGENTS.md`, and rule files and want a faster way to index, inspect, and maintain them.

Why it exists

Skills sprawl is manageable when the disk stays primary and the interface stays sharp.

Scan

Normalize scattered skill layouts into one index.

The app walks known directories, parses frontmatter or headings, resolves real paths, and builds a fast local index without taking ownership away from your files.

Search

Find a skill by name, description, or content.

SQLite-backed search keeps large local libraries usable without waiting on network services.

Edit

Write back to the source file directly.

No export step, no shadow copy, and no secondary format to keep in sync.

Portable

Built for local-first workflows.

The desktop app stays focused on local files, instant feedback, and minimal ceremony.

FAQ

Operational details before you install.

Why does macOS say the app is damaged?

The current macOS package is not notarized yet. Gatekeeper treats that as an untrusted unsigned build. Use Finder right-click Open or remove the quarantine attribute manually after you move the app into Applications.

Does the app keep a second copy of my skills?

No. Disk files remain the source of truth. The local SQLite database is only an index for search and filtering.

Is there a hosted sync service?

Not in this release. The current scope is local-first desktop management of skill files that already exist on your machine.
